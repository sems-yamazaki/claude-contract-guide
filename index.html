Claude 契約判断ガイド API契約／アプリ契約の選定基準

想定読者：Claude の法人利用を検討中の意思決定者・実務担当者

§01本資料の構成

Claude の法人利用には「API契約」と「アプリ契約」の2種類が存在する。それぞれ用途・規模・顧客要件により契約方法が異なる。本資料は判定ツールに回答することで推奨ルートを自動表示する形式とし、初見者でも適切な選定に至れるよう構成した。

用語集
API

プログラムから Claude を呼び出す仕組み。自社製品へのAI機能組み込み時に利用する。

アプリ（Claude.ai / Claude Code）

Claude.ai＝ブラウザ経由のチャット。Claude Code＝コマンドライン経由の開発支援。人間が直接利用する側。

Self-serve（セルフサーブ）

営業に連絡せず、Webでクレジットカード登録するだけで即日利用開始できる契約方式。Team・Enterprise の両方に self-serve 形態がある。

Enterprise（エンタープライズ）

大規模・高度なセキュリティ要件向けの上位プラン。2026年2月以降、Web即日の「self-serve」と営業経由の「sales-assisted」の2形態に分かれる。席単価$20＋API従量課金。

Enterprise self-serve / sales-assisted

self-serve＝Web即日契約でSCIM・監査ログ・Compliance APIまで利用可（クレカ/ACH・年払い）。sales-assisted＝営業経由で請求書払い・MSA・コミット割引・HIPAA・トライアルに対応。

Team プラン

アプリ向けの法人プラン。5〜150席の範囲で契約可能。内部に Standard 席と Premium 席が存在する。SCIM・監査ログは非対応。

Standard 席 / Premium 席

Team プラン内の席種。Standard＝チャット用（年払い月3,000円／月払い3,750円）／Premium＝開発者用（年払い月15,000円／月払い18,750円、Claude Code 利用可）。

PoC（ピーオーシー）

Proof of Concept ＝ 本格導入前の小規模なお試し・技術検証。

SLA（エスエルエー）

「99.9%以上稼働」のような稼働率保証契約。停止時には補償が発生する。

ZDR（ゼロデータリテンション）

リクエストデータを Anthropic に一切保存しない契約オプション。機密情報を扱う際に必須。

BAA / HIPAA

医療データを扱うための追加契約および米国法令。医療・保険・薬剤系の顧客で必須。

Tier（ティア）

API利用上限の段階。累計利用額に応じて自動昇格する（Tier 1〜Tier 4）。

SCIM（スキム）

入退社時にユーザーアカウントを自動で作成・削除する仕組み。Team では非対応で、Enterprise（self-serve / sales-assisted どちらでも可）が必要。

§02契約ルート判定ツール

API契約とアプリ契約は判定観点が異なるため、タブで分離している。双方に回答することで、それぞれの推奨ルートを把握できる。

タブ切替で API ／ アプリ それぞれの判定を実施

自社製品・システムへ Claude を組み込む場合の判定。全8問。

Q1API の用途は？ Claude を呼び出した API の主要な利用形態を選択する。
Q2扱うデータの機密度は？ API に渡すプロンプト・データの中で最も機密性が高いものを基準とする。
Q3顧客に「○○%稼働」のような稼働保証（SLA）を契約で明示する？ 「月99.9%稼働を保証・停止時は補償する」といった数値保証を顧客との契約書に明記するかどうか。社内専用の場合は「不要」を選択。
Q4将来の顧客展開・拡大予定は？ 現時点では小規模でも、将来的に対象となり得る顧客層を想定する。
Q5月のAPI利用見込みは？ 本番運用が安定した後の想定額。導入初期は小規模でも問題なし。
Q6エンドユーザー（最終的に AI 応答に触れる人）の規模は？ 外販なら顧客側の利用人数の見込み。
Q7AI 機能の業務クリティカル度は？ AI が誤応答・停止した時の業務影響度。
Q8既存の AWS / GCP 契約は？ AWS Bedrock や GCP Vertex AI 経由で Claude を利用するルートが存在する。

社員が Claude.ai／Claude Code を直接利用する場合の判定。全8問。

Q1使う人数は？ 開発者＋一般利用者の合計人数。
Q2Claude Code を使う開発者は存在するか？ Claude Code はコマンドラインで開発支援するツール。Premium 席が必要となる。
Q3SSO（シングルサインオン）の要件は？ Google Workspace や Azure AD によるログイン統一の要否。
Q4支払い方法の要件は？ 経理部門の要件により決定されるケースが多い。
Q5HIPAA・BAA は必要か？ 医療情報を Claude に入力する可能性があるなら必要。
Q6利用者が Claude に入力するデータの機密度は？ 想定される最高機密レベル。
Q7監査ログ・利用履歴の保管は必要か？ ISMS・社内コンプライアンスで誰がいつ何を入力したか追跡が必要かどうか。
Q8導入時期・スピード感は？ いつから使い始めたいか。

§03そもそも Claude の契約構造

Claude を法人利用する場合、契約は「API契約」と「アプリ契約」の2階建て構造となる。両方を利用する場合は、それぞれ個別に契約を締結する必要がある。

  • API契約：プログラムから Claude を呼び出し、自社製品に組み込む形態
  • アプリ契約：従業員が Claude.ai／Claude Code を直接利用する形態
  • 請求・ユーザー管理・料金体系は完全に独立（同一の Anthropic アカウント内でも別管理）

Standard 席（チャット用）

月 3,750 JPY（年払いで 3,000 JPY）

Premium 席（開発者向け）

  • Standard で提供される全機能
  • Claude Code 利用可（コード生成・編集・実行）

→ エンジニア・開発者向け

月 18,750 JPY（年払いで 15,000 JPY）

5席構成（最低契約）のコスト試算

支払い方式月額年額

Team プランの基本仕様

  • 5〜150席の範囲で契約可（最低5席、上限150席）
  • 支払い方式：クレジットカードのみ（請求書払い不可）
  • SSO：基本対応（Google Workspace / Azure AD 等）
  • SCIM：不可（必要なら Enterprise）
  • 監査ログ・Compliance API：不可（必要なら Enterprise）
  • HIPAA/BAA：不可（必要なら Enterprise sales-assisted）
  • 適格請求書発行（登録番号 T7700150134388・消費税10%別途）

Enterprise アプリ契約は2形態（2026年2月〜）

従来 Enterprise は営業経由のみだったが、2026年2月以降は Web から即日契約できる self-serve 形態 が追加された。SCIM・監査ログ・Compliance API といった主要なセキュリティ機能は self-serve でも利用できる。営業（sales-assisted）が必須となるのは請求書払い等の一部のみ。

Enterprise（self-serve・Web即日）

  • Web から即日契約・SSO設定・メンバー招待が可能
  • SCIM 自動プロビジョニング
  • 監査ログ・Compliance API
  • カスタムデータ保持・ロールベースアクセス
  • 支払い：クレジットカード／ACH（年払い）

→ SCIM・監査ログは欲しいが、クレカ払いで標準条件なら最速ルート

席単価 月3,000円（$20/席・年払い）＋ API 従量課金

Enterprise（sales-assisted・営業経由）

  • self-serve の全機能に加え：
  • 請求書払い / net terms
  • MSA・PO・非標準契約条件
  • コミット枠による割引・トライアル
  • HIPAA-readiness・専任サポート・AWS Marketplace 購入

→ 請求書払い・カスタム契約・HIPAA が必要なケース

席単価＋API 従量課金。割引・条件は商談で個別交渉
営業（sales-assisted）が"本当に"必要な条件
  • 請求書払い（net terms）が経理要件で必須
  • MSA・PO・非標準契約条件が必要

§05API契約の詳細

API契約の2つのルート

Self-serve（Web経由）

  • 即日利用開始可能
  • クレジットカード払い・従量課金
  • 段階的 Tier 昇格（後述）
  • SLA 保証なし・ZDR 不可・BAA 不可

→ 社内専用・PoC・小規模利用向け

Enterprise（営業ルート）

  • 契約締結まで 2〜12週間
  • 請求書払い・コミット枠による割引
  • SLA 数値保証・ZDR・BAA 締結可能
  • カスタム条件・専任サポート

→ 外販組込・大規模・規制業種向け

Self-serve の Tier 制度

Self-serve では累計クレジット購入額に応じて Tier が自動的に昇格する仕様。契約初期に大量利用することはできない。

Tier必要な累計購入額月の利用上限用途イメージ
Tier 1750 JPY15,000 JPY初回起動・PoC
Tier 26,000 JPY75,000 JPY本格検証
Tier 330,000 JPY150,000 JPY小規模本番
Tier 460,000 JPY750,000 JPY中規模本番の標準
Tier 4超営業経由で個別交渉が必要
Tier 昇格の運用ポイント
  • 本番投入前に Tier 4 への昇格を完了させるのが定石
  • 社内検証期間中にクレジットの少額購入を継続的に実施し、段階的に昇格させる
  • 本番リリース時点で Tier 1（月15,000円上限）の場合、レートリミット超過によりサービス停止のリスクあり

商用組込に関する規約

Anthropic の公式契約書（Commercial Terms）にて商用組込は明示的に許可されている。

§06次のアクション

判定ツールの結果に応じて、以下4つのフローのいずれかを順番に実施する。各 Step は 「順番に上から」 実行することを前提とする。

フロー1：初動アクション（全員共通）

どの契約ルートを選ぶかを確定する 所要時間 10〜20分
  1. 判定ツールで API／アプリ 双方を回答する §02 の判定ツールで API側・アプリ側 両方のタブに回答し、推奨ルートを確認する。両方やることで全体像が見える。 所要 5〜10分
  2. 不明用語を確認する §01 の用語集を展開し、判定結果に出てくる単語（ZDR・SCIM・BAA・Tier 等）の意味を把握する。 所要 5分
  3. 該当するフロー（2〜4）に進む 判定結果が Team プラン または Enterprise（self-serve・Web即日） ならフロー2、API Self-serve ならフロー3、Enterprise（営業経由＝請求書払い・HIPAA・API Enterprise 等） ならフロー4 へ進む。

フロー2：Webセルフサーブ契約（Team / Enterprise self-serve）

Team か Enterprise self-serve か SCIM・監査ログ・Compliance API が不要なら Team（手順は下記）。SCIM・監査ログが必要だが請求書払い・HIPAA は不要なら Enterprise self-serve を claude.com/pricing から同様に Web 即日契約する。どちらも営業を介さず即日で開始できる。
claude.ai で即日契約・社員へ配布（Team プランの例） 所要時間 30分〜1時間／ 即日利用開始可
  1. アカウント作成 claude.ai にアクセスし、業務用メールアドレスでアカウントを作成する。 所要 5分
  2. Team プランへアップグレード Settings → Team → "Upgrade to Team" を選択する。 所要 5分
  3. 席数・席種の選定 最低5席から契約可。開発者には Premium 席、その他には Standard 席 を割り当てる。 所要 10分
  4. 法人カードで決済 クレジットカードを登録して決済する。請求書払いは Team プラン非対応のため必要なら フロー4 へ。 所要 5分
  5. メンバー招待 対象社員のメールアドレスを入力し招待を送信する。利用開始は招待受諾後すぐ。 所要 5〜10分

フロー3：API Self-serve 契約（API・Webセルフサーブ）

console.anthropic.com で即日 API 利用開始 所要時間 30分〜1時間／ 即日利用開始可
  1. Console アカウント作成 console.anthropic.com にアクセスし、業務用メールアドレスでアカウントを作成する。 所要 5分
  2. Organization 情報の入力 Organization 名および 法人番号（T+13桁） を入力する。後の適格請求書発行に必要。 所要 5分
  3. 初回クレジット購入で Tier 1 起動 法人カードを登録し、$5（750円）相当のクレジットを購入する。これで Tier 1（月利用上限 15,000円）が起動する。 所要 5分
  4. APIキー発行・PoC 開始 Console から APIキーを発行し、開発・PoC 環境で利用を開始する。 所要 10〜20分
  5. 段階的に Tier 4 まで昇格 本番投入前にクレジット追加購入（累計 60,000円）を行い、Tier 4（月利用上限 750,000円）まで段階的に昇格させる。 本番投入前に完了
